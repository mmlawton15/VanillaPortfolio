My Work

This section contains my projects, group projects, and any future personal projects I work on (I have a few ideas up my sleeve). These apps are fully functioning, click the images to visit the page! If it's a backend project, you will be redirected to the Github Repo.

Password Generator

The Password Generator is an app that I would use to generate a completely random password. This can be used by anyone that needs a password for any kind of application, and very effectively randomizes the characters selected for your password.

Work Scheduler

The Work Day Scheduler functions similarly to a calendar app, it allows the user to schedule their daily tasks by the hour and reflects the current date and time. The previous time is greyed out, the current time will be red, and the upcoming time will be green. This uses local storage to allow for tasks to persist after the browser has been closed. Users can edit, delete and create tasks on this page

The Sheet Show

Our second group project is probably my favorite one we have worked on so far. It was a full stack application that was based on creating chatacter sheets for Dungeons and Dragons. I know nothing about DnD, so I enjoyed working with team members that were incredibly passionate and knowledgeable on the topic.

Note Taker

The Notetaker app was our challenge we used to show our skills with Express.js. It's an application that allows users to create, edit and delete notes and these notes persist using a JSON database. It was designed to replace a paper notebook!

Budget Tracker

The Budget Tracker was our first exposure to Progressive Web Applications. We handled the backend logic, more specifically the offline capabilities. If a user loses internet access, the app will log their transactions using IndexedDB. Once they regain network access, Indexed DB will load these transactions into the app!

Weather Dashboard

The Weather Dashboard was our introduction to Server-Side APIs. we were tasked with building a weather app, where the user could search for a city and get the daily forecast, as well as the 5 day forecast. The users recent searches were then logged as buttons on the far right for quick access.

Regex

I created a tutorial for Regular Expressions, which are incredibly powerful tools used for data validation. It allows developers to match user input to match a specific Regex formula comprised of different letters, numbers and symbols. Regex allows for flexibility and efficiency when coding and looking for certain inputs from users.

E-Commerce Backend

For the ECommerce backend project, we were provided with a working Express.js API and needed to configure it to use Sequelize and interact with a MySQL database.

README Generator

The README Generator was our 9th challenge of the bootcamp where I created a command-line application to generate a README file for projects. I used Inquirer and Node.js to record user input and invoke the application, and ultimately generate the README file.

Social Network API

This project was focused on NoSQL, MongoDB and Mongoose. We were challenged with building an API for a social network web application where users can share their thoughts, react to their friends thoughts, add and delete friends, update and delete thoughts. It was cool to see the baseline of how some of the popular social media platforms today got started.

Employee Database

The Employee Database is a CMS application that I picture someone in HR using to quickkly sort through data. In this app, you can view all departments, employees, salaries. You can also create,update or delete a department, a role or an employee.

Code Quiz

The coding quiz is an app that quizzes players on a pre-identified set of questions. It runs on a timer that decreases with every incorrect answer, and with every correct answer you will get points. At the end, you are able to add your initials ad log your high score.

Gas Guru

Gas Guru was our first group project where we focused on building a interactive front end project. Our Idea was an app that lets you type in a zipcode, and the lowest gas prices would be presented to you (via an api). Based on that zipcode, you also go an alert if there was inclement weather so you'd know to drive safely.

Horiseon

This was our first project of the bootcamp, where we essentially debugged the CSS and HTML files to make the website fully functional. This included making the links click to the correct sections and adding alt descriptions for every image.
